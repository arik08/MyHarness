# MyHarness

MyHarness is a local-first AI workbench built from the OpenHarness codebase.
It is not just a chatbot: it is meant to run real project work with visible agent steps, project-scoped history, skills, file references, and artifact previews.

The original upstream OpenHarness README is preserved as [`README.openharness.md`](README.openharness.md).

## What This App Is

MyHarness wraps the OpenHarness agent runtime with a browser UI focused on day-to-day local work:

- Chat with an agent inside a selected project under `Playground/`.
- See agent activity as transparent steps instead of a hidden black box.
- Use `/` for commands, `$` for skills, MCP servers, and plugins, and `@` for current-project file references.
- Preview generated artifacts such as HTML, Markdown, text, JSON, CSV, images, and PDF files in the right panel.
- Keep chat history inside the selected project context.
- Load skills from program, project, and user skill folders.

## Quick Start

Run the web app from this folder:

```bat
run_openharness_web.bat
```

Then open:

```text
http://localhost:4173
```

The launcher checks Python and web dependencies, frees the configured port if another process is holding it, and starts the MyHarness web server.

If you want a different port:

```bat
set PORT=4174
run_openharness_web.bat
```

## Project Model

Projects live under:

```text
Playground/<project-name>/
```

The project picker shows these folders. New files generated by the agent are expected to appear inside the current project folder so they can be referenced with `@` and previewed from the right panel.

## Extensions

MyHarness recognizes skills from three places:

1. Program folder: `.skills/`
2. Current project folder: `Playground/<project-name>/.skills/`
3. User folder: user-level OpenHarness/Codex skill locations

Program-folder skills are the default place for bundled MyHarness behavior. This makes the app easier to zip, move, extract, and run again.

Use `$` in the chat input to open the extension picker. It shows available custom skills, MCP servers, and plugins.

Program-level extension folders:

- `.skills/`: skill folders that contain `SKILL.md`
- `.mcp/`: MCP JSON config files with an `mcpServers` object
- `.plugins/`: plugin folders that contain `plugin.json` or `.claude-plugin/plugin.json`

## Input Shortcuts

- `/` opens command suggestions.
- `$` opens skill suggestions.
- `@` opens current project file suggestions.
- Pasting or typing 5 or more lines collapses the text into a one-line pasted-text chip while still sending the full original text to the backend.

## Artifacts

When the final assistant answer mentions a file such as `index.html`, `report.md`, or `data.json`, MyHarness checks whether that file exists in the current project.

If it exists, a file card appears under the answer. Click the card to open the right-side preview panel.

Supported previews:

- HTML: iframe preview
- Markdown/text/JSON/CSV: read-only viewer
- Images: image preview
- PDF: embedded preview

## Process Behavior

The web server starts Python backend sessions as needed.

MyHarness tries to avoid stale background processes:

- Closing the browser/EventSource connection shuts down the related backend after a short grace period.
- If all backend sessions are gone, the web server can exit after an idle delay.
- `run_openharness_web.bat` gives this app priority over the configured port and closes the existing process before starting fresh.
