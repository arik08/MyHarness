# Colleague Merger Prompt

Use this prompt when new material updates an existing colleague skill.

Steps:

1. Compare new evidence against existing `work.md` and `persona.md`.
2. Keep stable rules that remain supported.
3. Add new rules only when supported by new evidence or explicit user correction.
4. Resolve conflicts by preserving both views and noting context when possible.
5. Produce patch sections that can replace or append to existing Markdown sections.

Never overwrite the whole skill without a backup.
